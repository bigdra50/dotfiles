## 複数の選択肢がある場合

1. まず選択肢を提示
2. ユーザーの選択を待つ: AskUserQuestionツールを使用
3. 承認後に実行

## CLIツールの優先順位

Bashコマンド実行時は、従来ツールよりRust製の高速な代替ツールを優先する。

| 用途             | 優先    | 代替（非推奨） |
| ---------------- | ------- | -------------- |
| ファイル検索     | `fd`    | `find`         |
| テキスト検索     | `rg`    | `grep`         |
| ファイル表示     | `bat`   | `cat`          |
| ディレクトリ一覧 | `lsd`   | `ls`           |
| テキスト置換     | `sd`    | `sed`          |
| diff表示         | `delta` | `diff`         |
| JSON処理         | `jq`    | -              |
| プロセス監視     | `procs` | `ps`           |
| ディスク使用量   | `dust`  | `du`           |

これらのツールがインストールされていない場合のみ従来ツールを使用。

## Output Style

### 作業完了時のASCII図解

実装完了・作業完了後、変更内容をASCII図で視覚的に示す。

作業内容に応じて適切な図を選択：

| 作業内容              | 図の種類         |
| --------------------- | ---------------- |
| API・処理フロー実装   | シーケンス図     |
| クラス・型の追加/変更 | クラス図         |
| モジュール構成変更    | コンポーネント図 |
| 状態管理実装          | 状態遷移図       |
| データ構造変更        | ER図             |
| パイプライン・CI/CD   | フロー図         |

図解のルール：

- ASCII文字のみ使用（`+`, `-`, `|`, `>`, `*` 等）
- 変更箇所が明確にわかること
- 過度に巨大な図は避け、要点に絞る

### ランダムASCIIアート

作業完了時、低確率（体感5回に1回程度）で遊び心のあるASCIIアートを添える。

方針：

- 図解の代わりではなく、図解に加えてオマケとして出す
- CLIカルチャーのミーム、名言、小ネタなど自由に
- 短く、3〜6行程度に収める
- 作業内容や状況に合ったものを選ぶとなお良い

>   -- npm, probably

   |___|

  ╰──────────────────╯
