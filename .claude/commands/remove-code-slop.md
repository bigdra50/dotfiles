# Remove Code Slop

AI生成された不要なコメント・XMLドキュメント（スラップ）を除去し、コードの可読性を向上させます。

## 対象パターン

以下のスラップを検出・除去します：

1. **自明なXMLドキュメント**: プロパティ名やメソッド名を繰り返すだけのXML
2. **コード直訳コメント**: コードを日本語/英語に翻訳しただけのインライン
3. **冗長なremarks**: 過剰な使用例やコードサンプル
4. **enum値の過剰説明**: 名前から明らかなenum値のXML
5. **未使用のログ定数**: インライン化できるログメッセージ定数

## 保持するコメント

以下は削除しません：

- **WHY**を説明するコメント（設計判断の理由）
- 外部仕様への参照（RFC、数式の出典）
- 警告・前提条件（副作用、必須条件）
- 複雑なアルゴリズムの概要説明

## ワークフロー

1. **探索**: Task(Explore)でスラップパターンを検出
2. **確認**: 発見されたスラップをユーザーに提示
3. **除去**: 承認後、対象ファイルを編集
4. **検証**: コンパイル確認（言語に応じて）

## 使用方法

```
/remove-code-slop
```

オプション：特定のディレクトリのみ対象にする場合

```
/remove-code-slop src/core
```

## 注意事項

- 大量のファイルを変更する可能性があるため、事前にgit commitを推奨
- 言語固有の確認（Unity: MCP、TypeScript: tsc等）は環境に応じて調整
- 削除判断が不明確な場合はユーザーに確認
