# 実装ノート (implementation-notes.html)

仕様に基づく実装タスク中は、実装が仕様からどう逸脱・解釈されているかをユーザーに伝えるため、プロジェクトルートに `implementation-notes.html` を作成・更新する（running ノート）。

## 記録項目

| 項目 | 内容 |
| ---- | ---- |
| Design decisions | 仕様が曖昧な箇所で行った選択 |
| Deviations | 意図的に仕様から外した箇所と、その理由 |
| Tradeoffs | 検討した代替案と、最終的に採用したものを選んだ理由 |
| Open questions | ユーザーに確認・修正してほしい点 |

## ルール

- ファイル名は `implementation-notes.html`（固定、HTML 形式）
- 配置はプロジェクトルート
- 作業の進行に合わせて随時追記・更新する
- HTML を選ぶ理由は表現力（表・折りたたみ・色分け等）を活かすため
- 実装完了時にユーザーへファイルの存在を知らせる

## 対象タスク

仕様（SPEC）が明示された実装タスク全般。
単発のバグ修正・タイポ修正・1行変更等の小さい作業は対象外。

## 試験運用中

このルールは試験的に運用している。
不要・過剰と判断したらファイルごと削除する。
